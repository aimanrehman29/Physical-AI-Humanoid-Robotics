ROS 2 ایک مڈل ویئر ہے جو روبوٹ کے سافٹ ویئر حصوں کو آپس میں بات کرنے دیتا ہے۔ تین بنیادی تصورات: **نوڈز** (پروسیس)، **ٹاپکس** (پب/سب)، اور **سروسز** (ریکویسٹ/ریسپانس)۔

## بنیادی تصورات
- **نوڈ**: ایک پروسیس جو ایک واضح کام کرتا ہے (کیمرہ ڈرائیور، پلانر)۔  
- **ٹاپک**: ایک نامیاتی چینل برائے ڈیٹا اسٹریم (مثلاً `/camera/color/image_raw`)؛ پبلشر بھیجتے ہیں، سبسکرائبر وصول کرتے ہیں۔  
- **سروس**: ایک اینڈپوائنٹ برائے ایک بار کی کال (مثلاً `/reset_pose`)؛ کلائنٹ ریکویسٹ بھیجتا ہے، سروس جواب دیتی ہے۔  
- **پیرامیٹرز**: رن ٹائم میں ٹیون ہونے والی قدریں (مثلاً زیادہ سے زیادہ رفتار)۔  
- **لانچ فائلز**: متعدد نوڈز کو کنفیگ کے ساتھ چلائیں۔

## کم از کم مثالیں (Python, rclpy)
**پبلشر (talker)**: ایک سادہ نوڈ جو String میسج ٹاپک پر بھیجتا ہے۔  
**سبسکرائبر (listener)**: وہی ٹاپک سن کر میسج لاگ کرتا ہے۔  
**سروس**: `reset_pose` جیسی سروس جو SetBool ریسپانس دیتی ہے۔  
**لانچ**: talker + listener کو ایک لانچ فائل سے اکٹھا چلائیں۔

## گراف اور ٹولز
- `ros2 node list`, `ros2 topic list`, `ros2 interface show <msg>` گراف دیکھنے کے لیے۔  
- `ros2 topic echo <topic>` میسجز دیکھنے کے لیے۔  
- `ros2 run tf2_tools view_frames` ٹرانسفارمز دیکھنے کے لیے۔

## تیار ہونے کی چیک لسٹ
- [ ] نوڈ/ٹاپک/سروس کو ایک ایک جملے میں سمجھا سکتے ہیں۔  
- [ ] ایک سادہ پبلشر اور سبسکرائبر چلا سکتے ہیں۔  
- [ ] لانچ فائل سے متعدد نوڈز چلانا اور `ros2 topic list/echo` سے ٹاپکس دیکھ سکتے ہیں۔
