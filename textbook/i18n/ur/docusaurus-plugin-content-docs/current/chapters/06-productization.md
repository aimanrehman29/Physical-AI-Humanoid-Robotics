اس منصوبے میں آپ ادراک، ROS 2 میسجنگ، سمیولیشن، اور ایک سادہ ایکشن لوپ ملا کر ایک بنیادی اے آئی روبوٹ پائپ لائن بنائیں گے۔

## اہداف
- منظر میں ہدف آبجیکٹ کا پتہ لگانا۔  
- منصوبہ بول کر بتانا (“میں سرخ مگ ٹیبل سے اٹھاؤں گا”)۔  
- Gazebo/Isaac میں بازو/ایفیکٹر کو ہدف کی طرف حرکت دینا۔  
- نتیجے کو لاگ کرنا۔

## اعلی سطحی خاکہ
```text
Camera -> Perception -> Intent (JSON) -> Planner (ROS 2) -> Action (MoveIt/command) -> Feedback
```

## مراحل
1) **سمیولیشن**: Gazebo/Isaac میں میز اور سادہ آبجیکٹ کے ساتھ منظر لانچ کریں۔  
2) **ادراک**: تصویر لیں، ڈیٹیکٹر یا کیپشن ماڈل چلائیں؛ ہدف کا نام اور پوز حاصل کریں۔  
3) **Intent**: JSON اسکیمہ بھریں (`intent`, `target`, `location_hint`, `safety`)۔  
4) **ROS 2 نوڈز**:  
   - ادراک پبلشر: `/target_pose` پر ہدف کا پوز شائع کرے۔  
   - پلانر/ایکشن: پوز لے کر پہنچنے کا پلان بنائے اور ٹریجیکٹری بھیجے۔  
5) **نفاذ**: MoveIt یا سادہ جوائنٹ ٹریجیکٹری پبلشر سے بازو حرکت دیں۔  
6) **فیڈ بیک**: کامیابی/ناکامی لاگ کریں اور بول کر بتائیں۔  
7) **جائزہ**: لاگز/بیگز ری پلے کر کے ڈیبگ کریں؛ حفاظتی حدیں اور ادراک تھریشولڈ ایڈجسٹ کریں۔

## ٹارگٹ پوز میسج (مثال)
```text
std_msgs/Header header
string label        # "mug"
geometry_msgs/Pose pose
float32 confidence  # 0.0-1.0
```

## ٹیسٹ چیک لسٹ
- [ ] ہدف نظر آئے تو ادراک پوز شائع کرتا ہے۔  
- [ ] پلانر پوز لے کر ٹریجیکٹری بھیجتا ہے۔  
- [ ] بازو بغیر تصادم ہدف تک پہنچتا ہے۔  
- [ ] حفاظتی اسٹاپ/ابورٹ، قوت/رفتار کی حدیں مؤثر ہیں۔  
- [ ] لاگز/rosbags کامیابی سے ریکارڈ اور ری پلے ہوتے ہیں۔

## ڈیمو اسکرپٹ
1. منظر دکھائیں اور ہدف بتائیں۔  
2. پائپ لائن چلائیں؛ ادراک → intent → پلان → عمل بیان کریں۔  
3. حفاظتی اسٹاپ دکھائیں۔  
4. بیگ ری پلے کر کے مشاہدہ اور قابلِ سراغ دہی واضح کریں۔
