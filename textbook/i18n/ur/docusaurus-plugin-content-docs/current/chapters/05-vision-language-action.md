VLA وہ سلسلہ ہے جس میں روبوٹ دیکھتا ہے، سمجھتا ہے، فیصلہ کرتا ہے، اور عمل کرتا ہے۔ اسے سادہ رکھیں: **دیکھیں → سمجھیں → فیصلہ کریں → عمل کریں**۔

## ادراک سے زبان تک
- تصویر/ڈیپتھ فریم لیں → ڈیٹیکشن یا سیگمنٹیشن چلائیں → لیبلز/باؤنڈنگ باکس بنائیں۔  
- اختیاری: منظر کو کیپشن کریں (مثلاً “سنک کے بائیں ایک سرخ مگ”) تاکہ استدلال آسان ہو۔

## زبان سے عمل تک
- کمانڈز یا کیپشنز کو چند بنیادی مہارتوں میں نقش کریں: حرکت، گرفت، رکھنا، دیکھنا، بولنا۔  
- ایک واضح کمانڈ اسکیمہ رکھیں (intent + target + constraints) تاکہ ابہام نہ رہے۔

مثال اسکیمہ (JSON):
```json
{
  "intent": "pick",
  "target": "mug",
  "location_hint": "left of sink",
  "safety": {"max_force": 5, "speed": "slow"}
}
```

## عمل کا نفاذ
- موشن پلاننگ (MoveIt) سے پہنچنے/اٹھانے/رکھنے کی منصوبہ بندی کریں۔  
- ہدف نظر میں اور قابل رسائی ہو تو حرکت کریں؛ ادراک بدلنے پر دوبارہ پلان کریں۔  
- ٹائم آؤٹ اور سیف اسٹاپ رکھیں۔

## حفاظت اور گارڈ ریلز
- ہدف کی تصدیق (زبان یا بصری) “سرخ مگ اٹھاؤں؟ ہاں/نہیں”۔  
- انسانوں کے قریب قوت/رفتار محدود کریں؛ غیر متوقع رابطے پر فوراً رکیں۔  
- کیمرہ فریم، کمانڈز، اور اعمال لاگ کریں۔

## تیار ہونے کی چیک لسٹ
- [ ] “دیکھیں → سمجھیں → فیصلہ کریں → عمل کریں” ایک مثال کے ساتھ بیان کر سکتے ہیں۔  
- [ ] ایک سادہ کمانڈ اسکیمہ رکھتے ہیں۔  
- [ ] تصدیق اور حفاظتی حدیں نافذ کرتے ہیں۔
